# 用 AI Agent 辅助分析项目架构

> 本文档记录了"如何用 AI Agent 分析一个陌生项目"的实践案例。
> 这本身也是一种 AI 应用开发的能力——学会用 Agent 辅助工作。

## 场景

用户刚接手 Ascet Agent 项目，需要快速理解其底层框架 Pi 的架构。
直接逐文件读源码效率低，于是用 AI Agent（opencode 的 Task 工具）来辅助分析。

## Agent 提示词（Prompt）

以下是实际使用的提示词，这是一个**探索型子任务**的 prompt 模板：

```
我需要你帮我探索一个 TypeScript 项目 Pi 框架的结构，用于学习目的。
项目路径在 C:\...\pi-codex-pi-ascet-extension-prototype\

请帮我做以下事情（只做研究，不写任何代码）：

1. 读取 packages/agent/src/index.ts 和主要文件，了解 agent core 的核心架构
2. 读取 packages/ai/src/index.ts 和主要文件，了解统一 LLM API 是怎么设计的
3. 读取 packages/coding-agent/src/ 下的入口文件，了解 coding agent 怎么组织
4. 读取 packages/ascet-extension/ 下的 package.json 和主要源文件
5. 看看 packages/orchestrator/ 是干什么的
6. 看看 docs/ 目录下有没有架构文档

请返回一份结构化的报告，包含：
- Pi 框架的整体架构图（各包的职责和依赖关系）
- agent core 的核心概念
- LLM 统一 API 的设计模式
- Ascet extension 是如何扩展 Pi 的
- 对于学习者来说需要哪些前置知识
- 建议的学习路径

报告要详细但清晰，用中文写。
```

## 提示词设计要点（面试可用）

这个 prompt 体现了 AI 应用开发中的几个关键设计原则：

### 1. 明确任务边界
- "只做研究，不写任何代码"——限制 Agent 行为，避免意外修改
- 列出具体的文件路径，而不是泛泛地说"看看这个项目"

### 2. 结构化输出要求
- 明确要求返回"结构化报告"
- 列出报告应包含的 6 个部分
- 指定语言（中文）

### 3. 分步骤指令
- 6 个步骤有先后顺序（从核心包到扩展包）
- 每步都指向具体文件，降低 Agent 的探索成本

### 4. 目标导向
- "用于学习目的"——告诉 Agent 报告的受众和目的
- "建议的学习路径"——让 Agent 从学习者角度思考

> **面试金句**：如果面试官问"你怎么快速理解一个陌生项目"，可以说"我会用 AI Agent 辅助分析，给 Agent 结构化的探索指令，让它读源码后输出架构报告。这比手动逐文件读效率高很多。"

## Agent 输出结构（参考）

Agent 返回的报告结构如下，这是一个好的项目分析报告模板：

```
1. 项目概览（一句话定位 + 技术栈）
2. 整体架构图（ASCII 图 + 依赖关系）
3. 各模块职责表
4. 核心概念详解（按模块）
5. 关键设计模式
6. 前置知识清单
7. 建议学习路径（分阶段）
8. 关键文件速查表
9. 一句话总结
```

## 这个能力是怎么实现的

### Agent 的工作原理（预告版，W5 深入学）

opencode 的 Task 工具本质上就是启动了一个**子 Agent（subagent）**：

```
你（主对话）
  │
  │  "帮我探索 Pi 框架"（Task 工具调用）
  │
  ▼
子 Agent（独立上下文）
  │
  ├── 读文件 1 → 分析
  ├── 读文件 2 → 分析
  ├── 读文件 3 → 分析
  ├── ...
  │
  │  汇总分析结果
  │
  ▼
返回结构化报告给主对话
```

**关键点**：
1. **独立上下文**：子 Agent 有自己的对话窗口，不占用主对话的上下文
2. **工具调用**：子 Agent 能用 Read/Glob/Grep 等工具读文件
3. **自主决策**：子 Agent 自己决定读哪些文件、读多少、怎么组织报告
4. **单次返回**：子 Agent 只返回最终结果，中间过程不可见

### 对应到 Pi 框架的概念

这和 Pi 框架的 Agent 循环是同一个原理：

| 概念 | opencode Task | Pi Agent |
|------|--------------|----------|
| 工具调用 | Read/Glob/Grep | bash/read/edit |
| Agent 循环 | 读文件→分析→读更多文件 | prompt→LLM→工具→结果→下一轮 |
| 事件流 | （内部） | agent_start→turn→tool_execution→agent_end |
| 子 Agent | Task 工具 | Pi 的 subagents（ascet-extension 有9个） |

> **这就是 W5 要学的内容**：Function Calling 让 LLM 能调工具，Agent 循环让 LLM 能连续调多个工具完成复杂任务。你现在体验的"AI 帮你分析项目"就是 Agent 的典型应用。

## 实践建议

### 用 Agent 分析项目的 prompt 模板

```
我需要你帮我探索 [项目名] 项目的结构，用于 [目的]。
项目路径在 [路径]。

请帮我做以下事情（只做研究，不写任何代码）：
1. 读取 [核心文件1]
2. 读取 [核心文件2]
3. ...

请返回一份结构化的报告，包含：
- 整体架构图
- 各模块职责
- 核心概念
- 建议学习路径

报告用中文写。
```

### 注意事项
1. **大项目分批探索**：如果项目很大，分多次 Task，每次探索一个模块
2. **明确"不写代码"**：防止 Agent 修改源码
3. **给具体路径**：降低 Agent 探索成本，提高准确性
4. **要求结构化输出**：方便后续整理和复习
